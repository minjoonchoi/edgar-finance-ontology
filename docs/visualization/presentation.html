2. ODP 단계별 개발 과정

3단계 접근법

  1. 1단계 — 용어추출 (Term Extraction)

    데이터 원천, CQ, 분석 용어집에서 후보 용어 식별 및 정제

  2. 2단계 — 개념화 (Conceptualization)

    클래스 및 프로퍼티 정의, 설계 패턴 적용

  3. 3단계 — 형식화 (Formalization)

    OWL 제약조건 정의 (Cardinality, Disjoint, Key, PropertyChain)

2.1 1단계 — 용어추출 (1/2)

데이터 원천

  • SEC EDGAR Company Facts API
  • US-GAAP / IFRS Taxonomy (2024)
    • 회계 기준별 공식 태그 명칭 (예: us-gaap:Revenues, ifrs-full:ProfitLoss)
  • 기업 확장태그 (Extension)
    • ext: prefix 태그 포함. 회사별 비표준 항목 식별
  • Competency Questions (CQ)
    • "Which companies reported Revenue in FY2024?"
    • "What is the Operating Income of each REIT?"
    • "Which REITs derive revenue from rental income?"
  • 분석 용어집 (Analytical Vocabulary)
    • 회계·재무 분석 교재 및 산업 분류체계

2.1 1단계 — 용어추출 (2/2)

용어 정제 과정

  1. 텍스트 마이닝 기반 후보 식별
    • XBRL 태그 네임에서 의미 단위 추출
    • 명사 중심 후보 생성 (Revenue, OperatingIncome, Assets)
  2. 유의어·동의어 병합
    • SalesRevenueNet, Revenues, OperatingRevenue → Revenue
  3. CQ 및 자연어 패턴 매핑
    • 명사구 → 클래스 후보 (companies → LegalEntity)
    • 동사구 → 프로퍼티 후보 (reported → forMetric)

주요 참고 출처

US-GAAP/IFRS Taxonomy (FASB, IASB), Penman Financial Statement Analysis, Kieso Intermediate Accounting, Damodaran Investment Valuation, SIC/GICS/NAICS 산업 분류

2.1 CQ 및 자연어 패턴 매핑 예시

CQ: "Which companies reported Revenue in FY2024?"

  • [companies] → LegalEntity
  • [reported] → forMetric
  • [Revenue] → CanonicalMetric
  • [FY2024] → Period

CQ: "Revenue derived from rental income."

  • [Revenue] → CanonicalMetric
  • [rental income] → XbrlConcept
  • [derived from] → computedFrom

제약 문장: "Each observation has exactly one period."

  • → Restriction: (hasPeriod exactly 1 Period)

2.2 2단계 — 개념화 (Conceptualization)

핵심 설계 패턴

Observation Pattern

수치 데이터 = (주체, 속성, 맥락, 값) → FinancialObservation 중심 구조

Canonicalization Pattern

다양한 XBRL 태그를 CanonicalMetric으로 통합 (TagMapping, XbrlConcept 도입)

Contextualization Pattern

기간(Period), 통화(Currency), 회계기준(AccountingStandard)을 맥락으로 표현

Upper Ontology Reuse

LegalEntity ← fibo-be:LegalEntity
RegulatoryFiling ← fro:RegulatoryFiling

2.2 CQ 기반 클래스/프로퍼티 도출

CQ/자연어 예문 도출 요소 근거 설명
"Which companies reported Revenue?" LegalEntity, CanonicalMetric, forMetric "companies" 명사구 → 기업 클래스
"reported" 동사 → forMetric
"Revenue in USD during FY2024" hasCurrency, hasPeriod "USD" → 통화 속성
"FY2024" → 기간 속성
"Operating Income derived from GrossProfit minus SG&A" computedFrom 계산식 문장 구조로부터 도출
"Each observation has exactly one period" (hasPeriod exactly 1 Period) 자연어 제약 문장 패턴

2.3 3단계 — 형식화 (Formalization)

적용된 OWL 제약 패턴

유형 예시 표현 목적
Cardinality (forEntity exactly 1 LegalEntity) 필수 구성요소 보장
Functional FunctionalProperty(forMetric) 1:1 관계 강제
Disjoint DurationPeriod ⊥ InstantPeriod 시점/기간 혼동 방지
Key HasKey(FinancialObservation ...) 동일 조합 중복 방지
EquivalentClass RevenueObservation ≡ ... 추론기 자동 분류
PropertyChain (forEntity ∘ hasIndustry) ⊑ ... 산업 자동 상속

3.1 재사용 vs 자체 구축 클래스

재사용 클래스

  • efin:Company ← fibo-be:LegalEntity

    CQ "Which companies..."에서 [companies] 명사구

  • efin:RegulatoryFiling ← fro:RegulatoryFiling

    CQ "In which filings..."의 [filings] 명사구

핵심 자체 구축 클래스

  • FinancialObservation — Observation 패턴 중심
  • CanonicalMetric — CQ의 [Revenue], [Operating Income] 등 명사구
  • XbrlConcept — "tag", "concept", "XBRL element"
  • TagMapping — "map", "corresponds to" 구문
  • Period, Currency, AccountingStandard, Industry

3.2 핵심 오브젝트/데이터 프로퍼티

프로퍼티 Domain → Range CQ·자연어 근거
forEntity FinancialObservation → LegalEntity "Which companies..." 주어
forMetric FinancialObservation → CanonicalMetric "reported [Revenue]" 목적어
hasPeriod FinancialObservation → Period "in [FY2024]" 시간구
hasCurrency FinancialObservation → Currency "in [USD]" 화폐 명시
basedOnConcept FinancialObservation → XbrlConcept "tagged as [us-gaap:Revenue]"
computedFrom FinancialObservation → FinancialObservation "derived from Revenue – Expenses"
hasValue FinancialObservation → xsd:decimal "What was the [value]..."
hasSourceType FinancialObservation → xsd:string 소스 타입 (annual, composite, derived 등)
hasConfidence FinancialObservation → xsd:decimal 신뢰도 점수 (0.0-1.0)
hasReason FinancialObservation → xsd:string 선택 이유 설명
hasFiling FinancialObservation → RegulatoryFiling 관측값의 출처 문서
hasCIK LegalEntity → xsd:string SEC CIK 번호
hasSymbol LegalEntity → xsd:string 티커 심볼
hasName LegalEntity → xsd:string 회사명
fiscalYearEnd Period → xsd:string 회계연도 종료일 (MMDD 형식)
hasFormType RegulatoryFiling → xsd:string 제출 문서 형태 (10-K, 20-F 등)
hasAccessionNumber RegulatoryFiling → xsd:string SEC 접근 번호

3.3 제약(Constraints) 요약

제약 표현 CQ/자연어 근거
필수 키 (forEntity, forMetric, hasPeriod, hasCurrency) "Each observation must specify company, metric, period, and currency."
상호배타 DurationPeriod ⊥ InstantPeriod "Periods are either duration or instant, never both."
유일성 HasKey(FinancialObservation ...) "Only one observation per company per metric per period."
정의 클래스 RevenueObservation ≡ FinancialObservation ⊓ (forMetric value Revenue) "Revenue observations are those whose metric is Revenue."
체인 (forEntity ∘ hasIndustry) ⊑ hasIndustryOfObservation "An observation inherits the industry of its entity."

3.4 Canonical 지표 집합

카테고리 지표 CQ 및 자연어 근거
손익 (P&L) Revenue, OperatingIncome, NetIncome "What was Revenue of company X?"
"Operating Income by industry?"
현금흐름 (CF) CFO "Net Cash Provided by Operating Activities"
재무상태 (BS) Assets, Liabilities, Equity, CashAndCashEquivalents "What are total assets/liabilities at end of FY2024?"
파생 (Comparable) OperatingIncomeComparable "Comparable Operating Income across firms"

4. Protege 시연

온톨로지 시각화 및 추론

1. 스키마 구조 확인

  • 클래스 계층 구조: FinancialObservation, Period, CanonicalMetric 등
  • 프로퍼티 정의: forEntity, forMetric, hasPeriod, computedFrom
  • 제약 조건 검증: Cardinality, Disjoint, Key 제약 확인

2. 인스턴스 확인

  • 기업 데이터: S&P500 기업 정보 (CIK, 이름, 업종)
  • 재무 관측치: Revenue, OperatingIncome, NetIncome 등
  • 태그 매핑: XBRL 태그 → Canonical 지표 매핑 관계

5. 결론

핵심 성과

  • FIBO·FinRegOnt 최소 재사용 + Observation 패턴 중심 자체 구축
  • ODP 단계별 개발을 통한 체계적 설계
    • 용어추출: 회계·분석·산업 표준 용어 교차 검증
    • 개념화: 클래스·프로퍼티를 CQ 기반으로 체계화
    • 형식화: Reasoner 자동 분류 및 데이터 유일성 확보
  • 동일 재무지표의 이질적 태그를 Canonical 형태로 통합

달성 목표

  • XBRL 데이터의 의미적 정합성 (Semantic Consistency)
  • 투자자·분석가 질의의 재현성 (Query Reproducibility)
